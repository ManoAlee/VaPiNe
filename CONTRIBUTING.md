# Contributing to VaPiNe Sentinel

Welcome to the future of remote orchestration. We follow high-level engineering standards comparable to Tier-1 tech companies.

## 📜 Development Guidelines
1. **Clean Code**: Follow SOLID principles and maintain high modularity.
2. **Security First**: No feature shall bypass encryption or zero-trust protocols.
3. **Performance**: Latency must be measured and optimized in every PR.

## 🛡️ Coding Standards
- All core logic must be documented in `docs/architecture`.
- Use `HandshakeManager` for any new connection logic.

Thank you for helping us build the Sentinel era.
